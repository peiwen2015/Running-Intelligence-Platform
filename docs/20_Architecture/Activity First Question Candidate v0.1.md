# Activity First Question Candidate v0.1

## Purpose

This is not an Activity page design.

This is not an import flow design.

It only records the current product finding:

**the leading candidate for Activity's first question.**

## Context

Activity may not be the fifth page.

It may be the true entry point of the platform.

The runner's lived sequence is often:

```text
Garmin
  ↓
FIT
  ↓
Import
  ↓
Activity Review
```

This means Activity is not just one more view beside Weekly and Monthly.

It is the first place where the platform responds to a fresh run.

That also means Activity should not begin with:

- a metrics table
- a pace chart
- a detailed split review
- a generic coach comment

It should begin with the first question the runner truly wants answered after one session.

## Candidate Review

Three candidates are worth keeping for the first reduction.

### Q1

`這堂課，我真正練到了什麼？`

Strength:

- naturally centered on learning
- works for tempo, recovery, easy, long run
- aligns directly with Weekly's worldview

Risk:

- may feel abstract if the session was highly tactical

### Q2

`這堂課，真正完成了什麼？`

Strength:

- concrete
- easy to ground in one session
- works well for planned workouts

Risk:

- leans closer to completion than adaptation
- can drift toward task-checking

### Q3

`今天這次跑步，最值得記住的是什麼？`

Strength:

- warm
- memorable
- strongly reflective

Risk:

- feels more journal-like than coaching-first
- may fit a second-layer summary better than the opening question

## Current Read

Q1 is the leading candidate.

Not because it sounds the best.

But because it keeps Activity inside the same worldview that already shaped Weekly:

`Training → Adaptation → Learning`

If Weekly asks:

`這週，我到底練到了什麼？`

then Activity can naturally ask:

`這堂課，我真正練到了什麼？`

This creates a clean scale relationship:

- Activity = single-session learning
- Weekly = weekly learning

## What Q1 Changes

If Activity starts from Q1, the page is no longer centered on:

- whether the workout was completed
- whether pace hit the target
- whether the run looked impressive

It becomes centered on:

- what this session actually taught the body
- what kind of adaptation was created
- what was preserved even if the session was imperfect

That means Activity becomes different from Overview.

Overview answers:

`今天，我最該關心的是什麼？`

Activity would answer:

`這堂課，我真正練到了什麼？`

## Relationship to Other Pages

If this holds, the five major views stop feeling like five pages.

They become five coaching distances:

| Page | First question | Role |
| --- | --- | --- |
| Overview | 今天，我最該關心的是什麼？ | Attention |
| Activity | 這堂課，我真正練到了什麼？ | Single-session learning |
| Weekly | 這週，我到底練到了什麼？ | Weekly learning |
| Monthly | 我現在在哪？ | Position |
| Journey | 我正在成為什麼樣的跑者？ | Identity |

## Why Q2 Still Matters

Q2 should not be discarded yet.

It may still matter in Activity, but not necessarily as the opening question.

Possible role:

- Q1 defines the learning
- Q2 confirms what the session actually accomplished

In that sense, Q2 may survive later as a second-layer coaching statement.

## Validation

Q1 should remain in the lead only if it can naturally hold across at least:

1. a quality session
2. a recovery run
3. an easy run
4. a long run
5. an imperfect or interrupted session

If it cannot hold across those cases, then Activity has not yet found its first question.

## Next Step

Do not design Activity yet.

Do not sketch cards yet.

Use Q1 and Q2 to validate a few real single-session cases first.

If Q1 keeps making the reader nod before asking `為什麼？`, then Activity may have found its opening.
