# Activity Q1 Validation v0.1

## Purpose

This is not an Activity layout draft.

This is not an Activity writing draft.

This is only a validation slice for one question:

`這堂課，我真正練到了什麼？`

The goal is to test whether Q1 can stand on its own before any session page is designed around it.

## Why Activity Must Be Tested Differently

Activity sits directly behind Garmin and FIT import.

That means Activity is the first place where the platform must prove:

**this is not just another way to restate pace, HR, and training effect.**

So Activity Q1 should not be validated only by whether it sounds natural.

It should also be validated by whether it feels like coaching rather than Garmin.

## Validation Rule

Each case is allowed to answer Q1 in two sentences only:

1. not what was merely completed
2. what the runner or the body actually learned

Nothing else is added:

- no why
- no evidence
- no next step
- no UI

Q1 stays alive only if the reader:

1. nods before asking `為什麼？`
2. feels that this is **not** what Garmin would have said

## Validation Order

The hardest cases should go first.

Not because they are edge cases.

But because they expose the product worldview fastest.

### Case 1 | Disrupted Session

This should be the first test.

Examples:

- interval changed to easy
- tempo ended early
- heat forced a step back
- HR drift broke the original plan
- weather disrupted the workout

Why first:

If Q1 only works when the session looks successful, then Activity is not yet coach-like.

It is still just outcome commentary.

### Case 2 | Recovery Run

Why second:

Recovery is where the product proves that lighter sessions still contain learning.

### Case 3 | Easy Run

Why third:

Easy runs are common and often underinterpreted.

Q1 should still surface a real session learning even when the run was quiet.

### Case 4 | Quality Session

Why fourth:

Quality sessions are easy to make sound meaningful.

They are useful, but they should not be the first proof.

### Case 5 | Long Run

Why fifth:

Long runs often contain multiple possible learnings:

- endurance
- rhythm
- fueling
- restraint

This makes them a good later test for richness, but not the earliest test for validity.

## Example Direction

These are not final answers.

They only illustrate the kind of reading Q1 should support.

### Disrupted Session

**這堂課，你練到的不是完成原本的課表。**

**你練到的是：在狀態改變時，仍然把整體節奏留住。**

### Recovery Run

**這堂課，你練到的不是增加刺激。**

**你練到的是：讓前一段訓練真正被吸收。**

### Easy Run

**這堂課，你練到的不是跑得更快。**

**你練到的是：把節奏穩穩接回來。**

### Quality Session

**這堂課，你練到的不是單純達標。**

**你練到的是：在後半段仍然守住節奏。**

### Long Run

**這堂課，你練到的不是把距離撐完。**

**你練到的是：在疲勞開始出現後，仍然守住耐力主線。**

## What Failure Would Look Like

Q1 has not earned its place if the answers repeatedly drift into:

- pace summary
- target completion
- workout grading
- disguised Garmin metrics

Examples of weak readings:

- `這堂課完成了 8 公里 tempo。`
- `今天心率略高，但仍在合理區間。`
- `這次跑步整體表現不錯。`

These may be true, but they do not yet answer:

`這堂課，我真正練到了什麼？`

## Current Read

Activity looks promising not because it is easier than the other views.

It looks promising because its role is unusually clear:

- one session
- one immediate question
- one chance to show coaching where Garmin stops

## Next Step

Do not design Activity yet.

Use Q1 on a few real sessions, starting with a disrupted one.

If the sentence makes the reader nod and also think:

`這不是 Garmin，這比較像教練。`

then Activity may have found its opening.
