# Coach Letter Review v0.1

## Purpose

This is not a design document.

This is not a UX checklist.

This is a lightweight editorial review sheet for Monthly.

Its purpose is to help a fixed reader review the monthly letter in a repeatable way, so the page can mature through use rather than through repeated redesign.

## When to Use

Use this review after reading one Monthly page from start to finish.

Do not use it while editing.

Do not use it to predict what users might think.

Use it after an actual reading pass.

## Reviewer Role

The reviewer is not acting as a generic user.

The reviewer is acting as a **First Reader**:

- the same reader
- the same reading method
- different months
- comparable observations

## Review Questions

Each reading pass should answer the same five questions.

### 1. First Stop Point

Where did reading first slow down?

This may be caused by:

- unclear structure
- overly analytical tone
- repeated ideas
- internal framing mismatch

### 2. Easiest Skip

Which section was easiest to skim or skip?

This usually reveals one of two things:

- the section adds less than its screen space suggests
- the judgment was already complete before the section arrived

### 3. One Thing Remembered Five Minutes Later

After closing the page, what is the one idea that remains?

If nothing remains, the page may be clear but not memorable.

### 4. Coach Trust

After reading this page, how much does the coach still feel worth trusting?

Use only:

- `High`
- `Moderate`
- `Low`

Do not score this numerically.

This is an editorial judgment, not a survey metric.

### 5. What I Would Cut First

If one section had to be removed, what would go first?

This question is useful because weak sections often reveal themselves more clearly through subtraction than through critique.

## Review Template

```text
Coach Letter Review
Month:

First Stop Point:

Easiest Skip:

One Thing Remembered Five Minutes Later:

Coach Trust:

What I Would Cut First:
```

## What to Look For Across Multiple Months

Do not overreact to a single review.

Look for repeated patterns across several months.

Examples:

- if `Why` is repeatedly the first stop point, the explanation layer may be too analytical
- if `Evidence` is repeatedly the easiest skip, the page may be carrying too much supporting material
- if build months keep leaving the same memory, the page may need stronger editorial distinctiveness
- if trust drops while correctness stays intact, the issue may be voice, emphasis, or repetition rather than logic

## Success Condition

This review is doing its job when it helps answer:

- where reading friction repeats
- what the page consistently leaves behind
- whether the letter still feels like it comes from a credible coach

## What This Review Is Not For

This review is not for:

- visual polish decisions
- layout micro-adjustments
- technical QA
- proving architectural correctness

Its job is simpler:

to help Monthly mature as a letter.
