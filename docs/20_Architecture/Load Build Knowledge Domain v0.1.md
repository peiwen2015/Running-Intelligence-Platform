# Load Build Knowledge Domain v0.1

## Purpose

This is the second Coach Knowledge artifact.

Its job is not to define engine rules yet.

Its job is to define how a good running coach understands load build.

This document is the starting point for `Coach Knowledge Sprint 2 — Load Build`.

## Core Question

**How does a good coach distinguish productive stress from harmful stress?**

## Why Load Build Exists

Load build does not exist simply to make training bigger.

Load build exists to increase adaptation.

Its purpose is to expand what the runner can absorb, sustain, and repeat in the next phase of training.

In coaching terms, build is valuable only when added stress is still being converted into useful adaptation.

If added stress stops producing adaptation, it is no longer build.

## Load Build Types

### 1. Controlled Build

Stress increases while recovery still keeps pace.

The runner is carrying more work, but the training remains absorbable.

This is the ideal form of build.

### 2. Progressive Build

Each block rises a little relative to the last.

The increase is steady and cumulative rather than abrupt.

Its purpose is to create adaptation through continuity, not shock.

### 3. Aggressive Build

Stress rises quickly enough that adaptation may still occur, but risk is becoming more material.

This is not automatically wrong.

It is simply less forgiving.

### 4. Overload

Stress begins to outrun recovery capacity.

The runner may still complete the work, but the training is no longer clearly productive.

This is where the coach should start questioning whether continued increase is still worth it.

### 5. Unsustainable Build

The problem is no longer a single hard week.

The direction itself is becoming unrepeatable.

What is being built can no longer be carried forward into the next cycle.

## Signals

A coach may look for signals such as:

- load trend versus recent baseline
- distance trend versus recent baseline
- weekly rhythm stability
- long-run continuity
- quality-session distribution
- concentration of hard stress inside the week
- recovery completion from the previous block
- period-to-period training continuity

These signals do not explain build by themselves.

They only become meaningful when read together.

## Context

The same increase in stress can mean very different things depending on context.

Important build context includes:

- whether the previous block completed recovery successfully
- whether long runs remained stable while stress increased
- whether quality sessions were distributed or concentrated
- whether the runner is early, mid, or late in a build sequence
- whether the current rise follows a softer absorb phase
- whether the current phase demands growth, maintenance, or protection
- whether the increase is being repeated sustainably across periods

## Interpretation

The coach should not ask only:

`Is load up?`

The coach should ask:

`Is this increase still creating adaptation, or is it starting to create harm?`

That produces different interpretations:

| Load Build Type | Coach Interpretation |
|---|---|
| Controlled Build | The runner is taking on more work in a way that is still productive. |
| Progressive Build | The runner is accumulating adaptation through steady increases. |
| Aggressive Build | The runner is growing, but the margin for error is shrinking. |
| Overload | The runner is now carrying more stress than recovery is reliably absorbing. |
| Unsustainable Build | The runner is moving in a direction that cannot be repeated or carried forward safely. |

## Evidence Shape

The coach should be able to explain build using both signal and context.

Examples:

- `Load +22%` + `previous block recovered well` + `long run maintained` + `quality distributed`
  → likely Controlled Build

- `Load +10%` + `three consecutive blocks rising gradually` + `weekly rhythm stable`
  → likely Progressive Build

- `Load +25%` + `quality concentrated` + `recovery not clearly completed`
  → likely Aggressive Build or early Overload

- `Load +18%` + `long run dropped` + `quality stacked` + `fatigue not absorbed`
  → likely Overload

- `Load continues rising across periods` + `training rhythm becoming fragile`
  → likely Unsustainable Build

## What This Domain Does Not Do Yet

- It does not define engine rules.
- It does not define thresholds.
- It does not decide which coaching philosophy is universally correct.
- It does not assume all coaches interpret build the same way.
- It does not pretend FIT-only data can fully separate productive stress from harmful stress.

## Platform Position

This domain does not attempt to define the one true theory of build.

It defines what this platform chooses to treat as productive stress versus harmful stress.

That choice should remain explicit, reviewable, and evidence-driven.

## Observations

This section records structural observations only.

It does not conclude taxonomy yet.

### Observation 1

Load Build appears more directional than Recovery.

Recovery asks why stress is being reduced.

Load Build asks whether stress should continue moving forward.

### Observation 2

Load Build appears to depend more heavily on training structure than Recovery.

Distribution, sequencing, and continuity seem more central here than in simple recovery interpretation.

### Observation 3

Load Build appears more likely than Recovery to expose coaching philosophy differences.

Different coaches may accept different levels of aggressiveness before calling a block productive or harmful.
