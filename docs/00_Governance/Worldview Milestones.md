# Worldview Milestones

## Purpose

This document records how the worldview of the project changed over time.

It does not track feature delivery.

It does not track technical versions.

It records moments when the way we think about the product changed.

Release Notes answer:

`What changed?`

Worldview Milestones answer:

`Why did the meaning of the project change?`

## Why This Exists

Three years from now, the project may already contain:

- Journey
- Coach
- Digital Twin
- richer data sources
- stronger product surfaces

At that point, it may be easy to forget why those directions emerged.

This document exists to preserve the shifts in worldview that changed the product itself.

We did not arrive here only by adding features.

We arrived here because the way we understood the product changed.

```text
          Time

           ▲
```

Each milestone below marks a shift upward in abstraction.

## Milestone 1: Metadata First

We stopped designing tables.

We started governing meaning.

This milestone marks the birth of:

- Metadata Repository
- Metadata-first thinking
- governed definitions before implementation

The project stopped being a file conversion exercise.

It began becoming a governed data system.

## Milestone 2: Canonical Before Product

We stopped designing pages.

We started designing concepts.

This milestone marks the shift toward:

- Canonical Data Model
- Semantic Layer
- SQLite as an implementation of governed meaning

The project stopped asking:

`What pages do we want?`

It started asking:

`What concepts must exist before any page can be trusted?`

## Milestone 3: Coach Before Dashboard

We stopped asking:

`What data should we show?`

We started asking:

`What decision should we help make?`

This milestone marks the birth of:

- Product Design Principles
- coach-first language
- verdict before evidence
- state, reason, recommendation

The project stopped behaving like a report surface.

It began behaving like a coach workspace.

## Milestone 4: Journey Before Screens

We stopped designing pages.

We started designing a relationship.

This milestone marks the emergence of:

- Journey Product Vision
- continuity across Today, Weekly, Monthly, Shoes, Training, Metadata
- the product spine concept

The project stopped treating pages as isolated destinations.

It began treating them as chapters in one coaching relationship.

## Milestone 5: Time as Product

For most software,

time is only a filter.

For this platform,

time becomes part of the experience.

The product no longer organizes activities.

It organizes growth.

The product no longer measures running.

It accompanies change.

中文：

大部分產品把時間當成查詢條件。

而這個平台開始把時間設計成產品本身。

它整理的不再只是一次次活動。

而是一段段成長。

它衡量的不只是跑步。

而是改變。

This milestone marks the shift from:

`data over time`

to:

`understanding change through time`
